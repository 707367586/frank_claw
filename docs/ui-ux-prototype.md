# ClawX UI/UX 交互原型

**版本:** 4.0
**日期:** 2026-03-28
**对应 PRD:** v2.0
**对应架构:** v4.2

> Tauri + React + TypeScript 桌面交互范式。
> **核心变更：采用侧边栏 + 主内容区的两栏布局，侧边栏上部为 Agent 列表，下部为功能入口。点击 Agent 进入其对话视图。**

---

## 1. 信息架构

### 1.1 整体布局

两栏结构：左侧边栏（固定宽度 280px，可折叠） + 右侧主内容区。

```text
ZettClaw
├── 左侧边栏
│   ├── 上部（滚动区）
│   │   ├── + 新建 Agent
│   │   ├── 搜索Agent
│   │   └── Agent列表（按 Agent 分组，按时间排序）
│   │
│   └── 下部（固定底栏）
│       ├── 知识库
│       ├── 定时任务
│       ├── Connectors
│       ├── Agent & Skill
│       └── 用户头像 / 状态 / 设置
│
└── 主内容区
    ├── 对话视图（默认）
    ├── 知识库页面
    ├── 定时任务页面
    ├── Connectors 页面
    ├── Agent & Skill 页面
    └── 设置页面
```

### 1.2 侧边栏底部入口说明

| 入口 | 功能 | 阶段 |
|------|------|------|
| 知识库 | 知识源管理、索引状态、检索 | v0.1 |
| 定时任务 | time/event 触发任务、Run 历史 | v0.2 |
| Connectors | 飞书/Telegram 等外部渠道接入 | v0.2 |
| Agent & Skill | Agent 管理、Skills 市场、已安装列表 | v0.1（Agent）/ v0.2（Skill） |

### 1.3 导航显露原则

- `数据保险箱`：通过对话内操作或设置进入，不占一级入口
- `记忆中心`：通过 Agent 详情或对话右栏进入
- `产物`：内嵌在每个 Agent 页面的「产出」Tab 中，不独立成页

### 1.4 全局核心对象

| 对象 | 用户理解 | 默认入口 |
|------|----------|----------|
| Agent | 长期工作的 AI 角色/能力容器 | 侧边栏 Agent 列表 |
| Conversation | 一段可持续的对话线程 | 点击 Agent → 进入对话视图 |
| Run | 一次执行过程，可能包含多步 | 对话区内 / 定时任务详情 |
| Memory | 被持久化的偏好、事实、经验 | Agent 详情 / 对话右栏 |
| Source | 知识源和被引用证据 | 知识库 / 对话右栏 |
| Artifact | Agent 生成的文件或结果 | Agent 页「产出」Tab |
| Task | 后台可重复执行的任务定义 | 定时任务页 |

## 2. 页面层级总览

```text
侧边栏 Agent 列表 → 点击 Agent → Agent 页面（主内容区）
    ├─「对话」Tab
    │   ├─ 对话消息流
    │   └─ 可选右侧信息面板（来源/执行/审批/记忆）
    └─「产出」Tab
        ├─ 搜索产物
        └─ 产物列表（文档/代码/图片/记忆等）

侧边栏底部入口 → 各功能页面（主内容区全屏展示）
    ├─ 知识库：知识源列表 + 检索工作台
    ├─ 定时任务：任务列表 + Run 历史
    ├─ Connectors：渠道列表 + 路由配置
    ├─ Agent & Skill：Agent 管理 + Skill 市场
    └─ 设置：模型 / 安全 / 外观 / 健康
```

**设计要点：**

- **顶栏**：左侧 ≡ 汉堡菜单（折叠/展开侧边栏）、中部 Agent 名称 + 下拉（切换 Agent）、右侧设置齿轮 + 用户头像
- **Tab 栏**：「对话」和「产出」两个 Tab，紧贴顶栏下方
- **对话空状态（Agent 介绍页）**：Agent 图标 + 名称 + 描述 + 能力标签 + 建议提示词
- **输入栏**：左侧 `+` 附件按钮和 `Skills` 入口，右侧模型选择器 + 发送按钮
- 侧边栏以 Agent 为一级条目，每个 Agent 显示名称 + 状态
- Agent 列表按活跃状态排序：运行中 > 待审批 > 空闲
- 侧边栏底部四个功能入口固定不随滚动

### 3.1.1 侧边栏 Agent 列表

每个 Agent 条目显示：

| 元素 | 说明 |
|------|------|
| 状态指示器 | ● 活跃（运行中/待审批） / ○ 空闲 |
| Agent 名称 | 单行截断 |
| 状态描述 | 运行中 / 空闲 / 索引进度 / N 待审批 |

右键菜单：Agent 详情 / 新建对话 / 克隆 / 停用 / 删除。

搜索框支持按 Agent 名称、角色说明模糊匹配。

### 3.1.2 Agent 页面结构

点击侧边栏某个 Agent 后，主内容区展示该 Agent 的页面，包含两个 Tab：

**「对话」Tab（默认）**

新对话空状态为 Agent 介绍页：
- Agent 图标 + 名称 + 一句话描述
- 能力标签（如「对话」「文件处理」「任务规划」「代码生成」）
- 建议提示词列表（3-5 条，点击即发送）

进入对话后：
- 消息流区域，流式输出
- 可选右侧信息面板（按需展开）：来源 / 执行轨迹 / 审批 / 记忆

**「产出」Tab**

展示该 Agent 所有历史产出物，按时间倒序排列：

产出 Tab 设计要点：
- 顶部搜索栏，支持按文件名/内容模糊搜索
- 右侧筛选/排序切换（列表视图 / 网格视图）
- 每条产物显示：类型图标（文档/代码/图片）+ 标题 + 日期 + 摘要
- 图片类产物直接展示缩略图预览
- 点击产物打开详情预览 / 编辑 / 下载

**输入栏（对话 Tab 底部固定）**

| 区域 | 元素 |
|------|------|
| 左侧 | `+` 附件按钮（拖入或选择文件）、`Skills` 按钮（调用已安装 Skill） |
| 中部 | 文本输入框，支持多行 |
| 右侧 | 模型选择器下拉（如 Sonnet 4.6 ▾）、发送按钮 |

**顶栏**

| 区域 | 元素 |
|------|------|
| 左侧 | ≡ 汉堡菜单（折叠/展开侧边栏） |
| 中部 | 当前 Agent 名称 + ▾ 下拉（快速切换 Agent） |
| 右侧 | ⚙️ 设置入口、👤 用户头像 |

### 3.2 对话交互规则

对话主区为流式消息列表，右侧信息面板按需展开（默认隐藏，有内容时自动显示）。

- 检测到知识引用时自动显示 `来源` 面板
- 检测到多步执行时自动显示 `执行` 面板
- 有待确认动作时 `审批` 面板自动置顶并高亮
- 产物生成后 `产物` 面板显示文件列表
- `记忆` 面板只展示摘要，点击跳转记忆详情

### 3.3 对话消息中的执行可视化

在消息流中插入可折叠的结构化步骤卡：

```text
[执行中]
Step 1/4 读取 auth.ts, route.ts, session.ts
结果: 已定位 2 个重复校验逻辑
[查看详情]
```

`查看详情` 展示：输入来源、使用工具、风险说明、结果摘要、相关产物链接。不展示 raw reasoning。

### 3.4 Agent 详情抽屉

从 Agent 列表右键或顶栏 Agent 名称下拉中进入。

Tabs: 概览 / 配置 / 知识关联 / 记忆概览 / 权限档案(v0.2) / Skills(v0.2)

概览页包含：Agent 描述、能力标签、模型配置、创建时间、对话数、产出数。

**左栏 — 知识源列表**

| 元素 | 说明 |
|------|------|
| 搜索栏 | 按知识源名称模糊匹配 |
| 筛选 Tab | 全部 / 全局 / Agent 专属 |
| 知识源卡片 | 名称 + 关联范围（全局/某 Agent）+ 文档数量 + 索引状态 |
| 添加知识源 | 支持本地文件夹、单文件、URL 导入 |

知识源卡片右键菜单：编辑 / 重新索引 / 更改关联范围 / 解除关联 / 删除。

解除关联不删文件，只改检索可见性。

**右栏 — 检索工作台**

- 输入问题或关键词，返回命中的文档段落
- 每条结果显示：标题 + 命中段落摘要 + 来源知识源/文件名 + 「查看原文」链接
- 默认只展示"命中证据"和"是否可用于回答"
- 高级模式（右上角切换）才显示检索方式、相关度分数、chunk ID

**交互流程**

```text
添加知识源 → 选择来源（本地文件夹/文件/URL）→ 设置关联范围（全局/Agent 专属）
→ 开始索引（显示进度条）→ 索引完成后可检索
```

**空状态**

```text
还没有添加任何知识源。
添加文档、网页或文件夹，让 Agent 基于你的知识回答问题。
[添加本地文件]  [导入 URL]
```

**任务卡片**

| 元素 | 说明 |
|------|------|
| 触发类型图标 | ⏱ 定时触发 / 📡 事件触发 |
| 任务名称 | 用户自定义 |
| 关联 Agent + 触发规则 | 如「编程助手 · 每天 08:00」 |

## 4. 关键交互流程

### 4.1 首次启动引导

```text
启动 → 欢迎页 → 语言与外观 → 本地优先说明（数据留本机/未经允许不联网）
→ 配置模型（本地/云端/测试连接） → 创建首个 Agent → 进入对话
```

### 4.2 对话请求流程

```text
用户输入 → controlplane → memory recall + knowledge retrieval（并行）
→ 如需工具/写入/联网: Permission Gate → 审批单
→ 执行: 流式输出 + 结构化 Step + 右栏同步更新
→ DLP 扫描 → 结果完成
```

### 4.3 文件工作区边界

- 拖入外部文件 → 复制到 Workspace 受管目录 → Agent 处理受管副本
- 结果默认保存在 Workspace；导出到工作区外需确认

### 4.4 审批流程

```text
Agent 请求高风险动作 → 按动作分组展示审批单 → 用户逐项决定
（本次允许/本会话允许同范围/加入白名单/拒绝） → 记录审计 → 继续/中断
```

### 4.5 侧边栏导航流程

```text
点击侧边栏 Agent → 主内容区展示该 Agent 页面（默认「对话」Tab）
切换「产出」Tab → 查看该 Agent 的所有历史产出物
顶栏 Agent 名称下拉 → 快速切换到其他 Agent
点击侧边栏底部入口 → 主内容区切换到对应功能页
点击 + 新建 Agent → 主内容区显示创建面板 → 创建后自动进入 Agent 页面
```

## 6. 全局设计规范

### 6.1 导航与布局

| 元素 | 规范 |
|------|------|
| 顶栏 | 48px，≡ 折叠菜单 + Agent 名称下拉（居中）+ 设置/用户头像（右侧） |
| 侧边栏 | 固定宽度 280px，可通过 ≡ 折叠；上部 Agent 列表可滚动，下部功能入口固定 |
| 主内容区 | 自适应宽度，Agent 页面含「对话」「产出」Tab 切换 |
| 输入栏 | 对话 Tab 底部固定，含附件/Skills/文本输入/模型选择/发送 |
| 右侧信息面板 | 对话中按需展开，默认 320px |
| 抽屉 | Agent 详情、记忆详情等优先用右侧抽屉 |
| Modal | 仅用于审批、删除、创建等需显式确认的动作 |

### 6.2 信任与透明

- 不展示 raw chain-of-thought，只展示结构化步骤摘要
- 来源始终可追溯，知识引用能回到原文
- 审批必须列出路径、域名、范围、风险原因
- 高风险动作后可直接跳审计记录

### 6.3 渐进披露

| 默认态 | 高级态 |
|--------|--------|
| 命中证据摘要 | 检索分数 / chunk ID / 检索方式 |
| 记忆摘要 | JSON 原文 / 访问频次 / 鲜活度 |
| 权限范围 | 能力向量、审计事件 ID |
| 系统状态提示 | CPU / 内存 / 模块状态详情 |

### 6.4 状态反馈

| 状态 | 表达方式 |
|------|----------|
| 正常空闲 | 不常驻提醒 |
| 后台忙碌 | 侧边栏 Agent 项显示运行指示器 |
| 需审批 | Agent 项徽标 + 审批面板高亮 |
| 出错 | 页面内联错误 + 可恢复操作 |
| 成功完成 | Toast + 可跳转结果 |

### 6.5 快捷键

| 快捷键 | 动作 |
|--------|------|
| `⌘K` | 全局搜索 / 命令 |
| `⌘N` | 新建对话 |
| `⌘⇧N` | 新建 Agent |
| `⌘\` | 折叠/展开侧边栏 |
| `⌘.` | 中断当前执行 |

### 6.6 空、载入、错误状态

- **空状态**：必须给下一步动作
- **载入状态**：骨架屏或步骤进度
- **错误状态**：必须给恢复动作
- **后台处理中**：轻状态条或内联卡

### 6.7 视觉规范

专业、克制、可信。深色为主色调（参考截图），安全/审批用低饱和功能色。标题/正文/辅助/代码四层字体。统一圆角。阴影仅用于浮层。动效重点在面板展开、步骤插入、状态切换。

## 8. 结论

UI 主轴：

`侧边栏选 Agent → 对话 Tab 交互 → 产出 Tab 查看产物 → 侧边栏切换到知识库/任务/设置`

用户在三个层面持续建立信任：

1. **知道正在发生什么**
2. **知道为什么需要确认**
3. **知道出了问题如何追溯和恢复**
