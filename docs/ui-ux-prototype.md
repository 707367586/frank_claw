# ClawX UI/UX 交互原型

**版本:** 2.1 (Revised)  
**日期:** 2026-03-23  
**对应 PRD:** v2.0  
**对应架构:** v4.2

> 本版原型按 `PRD v2.0`、`architecture v4.2`、`autonomy-architecture v3.1` 对齐，明确采用  
> **Tauri + React + TypeScript** 的桌面交互范式，而不是早期的 SwiftUI 叙事。

---

## 1. 设计重构原则

### 1.1 本次调整的核心判断

1. **工作优先，不是系统地图优先**
   默认首页应展示“正在推进什么、卡在哪里、最近产出了什么”，而不是把所有子系统平铺给用户。

2. **阶段显露，不提前暴露后续能力**
   `v0.1` 只暴露本地闭环所需入口；`v0.2` 才展开主动任务、渠道、Skills；`v0.3+` 再引入产物、账号、同步等平台层能力。

3. **透明执行，不暴露 raw chain-of-thought**
   对话页展示结构化执行轨迹、来源引用、审批记录和产物，不展示“思考链”原文。

4. **默认安全，复杂控制按需展开**
   默认只给用户看“影响范围、风险原因、允许范围”；高级策略和技术细节进入“查看详情”。

5. **把 Agent 当作工作容器，而不是首页主角**
   Agent 仍是核心对象，但应服务于“会话、运行、任务、产物”这些更贴近用户结果的对象。

### 1.2 设计关键词

- `Workspace-first`
- `Run-visible`
- `Trust-by-default`
- `Progressive disclosure`
- `Keyboard-first`
- `Local-first`

## 2. 信息架构

### 2.1 v0.1 导航结构

`v0.1` 只保留 6 个一级入口：

```text
ClawX.app
├── 侧边栏
│   ├── 工作台        ← 首页，继续工作 / 待处理 / 最近产物 / 常用 Agent
│   ├── 对话          ← 当前上下文内的会话与执行
│   ├── 记忆中心      ← 用户记忆 / Agent 记忆 / 审计
│   ├── 知识库        ← 知识源管理 / 检索 / 引用
│   ├── 数据保险箱    ← 工作区版本点 / 差异 / 回滚
│   └── 设置          ← 模型 / 安全 / 外观 / 语言 / 健康
│
├── 顶部栏
│   ├── 全局搜索 / 命令面板 (⌘K)
│   ├── 当前上下文标题
│   ├── 服务状态徽标   ← 仅在异常或后台忙碌时高亮
│   └── 用户菜单
│
└── 主内容区 + 可选右侧信息面板
```

### 2.2 v0.2 导航扩展方式

`v0.2` 不新增 3 个分散一级入口，而是在侧边栏中加入一个收纳型分组 `扩展执行`：

```text
ClawX.app
├── 工作台
├── 对话
├── 记忆中心
├── 知识库
├── 数据保险箱
├── 扩展执行
│   ├── 任务
│   ├── 渠道
│   └── Skills
└── 设置
```

这样可以避免首页像“后台控制台”，同时符合 `v0.2` 的能力边界。

### 2.3 v0.3+ 导航显露原则

- `产物管理`：当产物规模和跨 Agent 检索需求明显增加后，才提升为独立入口。
- `设备管理`：仅在 `v0.5` 物理 Agent 能力启用后显露。
- `账号 / 备份 / 社区`：保留在设置内，默认不侵入本地闭环主路径。

### 2.4 全局核心对象

| 对象 | 用户理解 | 默认入口 |
|------|----------|----------|
| Agent | 一个长期工作的 AI 角色/能力容器 | 工作台 |
| Conversation | 一段可持续的对话线程 | 对话 |
| Run | 一次真实执行过程，可能包含多步 | 对话 / 任务详情 |
| Memory | 被持久化的偏好、事实、经验 | 记忆中心 |
| Source | 知识源和被引用证据 | 知识库 / 对话右栏 |
| Artifact | Agent 生成的文件或结果 | 对话右栏 / 工作台 / v0.3 产物页 |
| Task | 后台可重复执行的任务定义 | v0.2 扩展执行 |

### 3.1 Agent 详情不再占一个一级导航位

Agent 详情从“独立页面主路径”调整为“上下文内抽屉 / 面板”：

- 从工作台卡片进入 Agent 详情抽屉
- 从对话页标题进入 Agent 设置面板
- 内容 Tabs:
  - `概览`
  - `配置`
  - `知识关联`
  - `记忆概览`
  - `权限档案 (v0.2)`
  - `Skills (v0.2)`

这样保留 PRD 所需内容，但不打散主导航。

- 帮用户回到未完成工作
- 暴露需要用户决策的节点
- 让后台运行过程有存在感，但不打扰
- 保留创建 Agent 的入口，但不让 Agent 卡片占满整个屏幕

### 4.3 对话页交互规则

- 默认只展开 `来源` 面板。
- 检测到多步执行时，自动显示 `执行` 面板。
- 有待确认动作时，`审批` 面板自动置顶并高亮。
- 产物生成后，`产物` 面板显示文件列表和预览入口。
- `记忆` 面板默认展示摘要，不展示底层 JSON。

### 4.4 对话消息中的执行可视化

不把执行信息完全塞在右侧栏，也不暴露思考链原文，而是在消息流中插入可折叠的结构化步骤卡：

```text
[执行中]
Step 1/4 读取 auth.ts, route.ts, session.ts
结果: 已定位 2 个重复校验逻辑
[查看详情]

Step 2/4 比对 register.ts 实现
结果: 可复用 session adapter 模式
[查看详情]
```

`查看详情` 展示：

- 输入来源
- 使用工具
- 风险说明
- 结果摘要
- 相关产物链接

不展示 raw reasoning 文本。

## 5. 关键交互流程

### 5.1 首次启动引导

首次引导从“先配模型”改成“先建立信任边界，再进入工作”：

```text
启动 App
  ↓
欢迎页
  ↓
语言与外观
  ↓
本地优先说明
  - 数据默认留在本机
  - 未经允许不联网
  - 工作区内才支持回滚
  ↓
选择工作区位置
  ↓
配置模型
  - 本地模型
  - 云端 Provider
  - 测试连接
  ↓
创建首个 Agent
  ↓
进入工作台
```

### 5.2 对话请求流程

```text
用户输入文本 / 文件
  ↓
controlplane → clawx-api → clawx-runtime
  ↓
并行:
  - memory recall
  - knowledge retrieval
  ↓
如需工具/写入/联网:
  - Permission Gate 评估
  - 需要确认则弹出审批单
  ↓
进入执行
  - 流式输出回答
  - 插入结构化 Step Summary
  - 右栏同步更新来源 / 执行 / 审批 / 产物 / 记忆
  ↓
DLP / 安全扫描
  ↓
结果完成
```

### 5.3 对话内创建任务（v0.2）

```text
用户: 帮我每天早上 8 点检查 AI Safety 新论文并发飞书
  ↓
系统解析为结构化任务
  - 名称
  - Goal
  - Trigger = time
  - 通知方式 = 飞书
  - 首次执行权限范围
  ↓
展示确认卡片
  ↓
用户确认
  ↓
创建 Task + Trigger
  ↓
任务页可见，上次/下次执行时间立即可见
```

### 5.4 审批流程

```text
Agent 请求高风险动作
  ↓
按动作分组展示审批单
  ↓
用户逐项决定
  - 本次允许
  - 本会话允许同范围
  - 加入白名单后允许
  - 拒绝
  ↓
系统记录审计
  ↓
执行继续 / 中断
```

### 5.5 回滚流程

```text
打开数据保险箱
  ↓
按任务浏览版本点
  ↓
预览差异
  ↓
选择粒度
  - 单文件
  - 文件夹
  - 整个任务
  ↓
确认回滚
  ↓
自动创建当前版本点
  ↓
执行回滚并展示结果
```

### 5.6 知识库索引流程

```text
添加知识源
  ↓
FSEvents 监控
  ↓
解析 / 分块 / 建索引
  ↓
工作台显示后台进度
  ↓
完成后可在知识库页与对话页引用
```

## 7. 全局设计规范

### 7.1 导航与布局

| 元素 | 规范 |
|------|------|
| 侧边栏 | `v0.1` 固定 6 项；`v0.2` 增加一个“扩展执行”分组 |
| 顶部栏 | 48px，高频动作只保留 `⌘K`、当前上下文、状态反馈 |
| 右侧信息面板 | 默认 320px，按数据有无自动显隐 |
| 抽屉 | Agent 详情、记忆详情、文档详情优先用右侧抽屉，而不是新页面 |
| Modal | 仅用于审批、回滚、删除、创建等需要显式确认的动作 |

### 7.2 信任与透明

| 原则 | 规则 |
|------|------|
| 不展示 raw chain-of-thought | 只展示结构化步骤摘要 |
| 来源始终可追溯 | 对话中的知识引用需能回到原文 |
| 审批必须具体 | 必须列出路径、域名、动作范围、风险原因 |
| 审计默认可见 | 高风险动作后的结果页可直接跳审计记录 |

### 7.3 渐进披露

| 默认态展示 | 高级态展示 |
|-----------|-----------|
| 命中证据摘要 | 检索分数 / chunk ID / 检索方式 |
| 记忆摘要 | JSON 原文 / 访问频次 / 鲜活度细节 |
| 权限范围 | T1/T2/T3、能力向量、审计事件 ID |
| 系统状态提示 | CPU / 内存 / 模块状态详情 |

### 7.4 状态反馈

| 状态 | 表达方式 |
|------|----------|
| 正常空闲 | 不常驻大面积提醒 |
| 后台忙碌 | 顶栏轻量状态徽标 + 工作台卡片 |
| 需审批 | 工作台待处理卡 + 对话页审批面板高亮 |
| 出错 | 页面内联错误 + 可恢复操作 |
| 成功完成 | Toast + 可跳转到相关结果 |

### 7.5 快捷键

| 快捷键 | 动作 |
|-------|------|
| `⌘K` | 全局搜索 / 命令 |
| `⌘N` | 新建对话 |
| `⌘⇧N` | 新建 Agent |
| `⌘1-6` | 切换一级导航 |
| `⌘.` | 中断当前执行 |

| 页面 / 模块 | 包含内容 |
|------------|----------|
| 工作台 | 继续工作、待审批、运行中、最近产物、常用 Agent |
| 对话 | 会话列表、流式输出、结构化执行轨迹、来源引用、审批、记忆摘要 |
| 记忆中心 | 用户记忆 / Agent 记忆浏览、编辑、固定、删除、审计 |
| 知识库 | 知识源管理、索引进度、检索工作台、原文定位 |
| 数据保险箱 | 版本点、差异预览、任务级 / 文件级回滚 |
| 设置 | 模型、安全基线、外观、语言、健康 |
| 全局 | ⌘K、首次引导、审批弹窗、基础状态反馈 |

### 8.2 v0.2 扩展执行层

| 页面 / 模块 | 包含内容 |
|------------|----------|
| 扩展执行 | 任务 / 渠道 / Skills 三个 Tabs |
| 任务 | `time/event` 触发、Run 历史、反馈统计、冷却与抑制 |
| 渠道 | 渠道 CRUD、路由配置、连接状态 |
| Skills | 已安装列表、签名状态、执行级别、启停、配置 |
| Agent 详情增强 | 权限档案、自动放行范围、最近一次降级原因 |
| 通知 | 任务结果通知、反馈按钮、抑制策略可见 |

### 8.3 v0.3+ 平台服务层

| 页面 / 模块 | 显露策略 |
|------------|----------|
| 产物管理 | 当跨 Agent 产物规模上升后再独立成页 |
| 用量统计 | 放入设置，不进入主导航 |
| 账号 / 备份 / 迁移 / 更新 | 仍在设置内，避免侵入主路径 |
| 设备管理 | `v0.5` 后独立显露 |
| 社区 / 商店 | 不影响本地闭环默认体验 |

### 8.4 明确不应提前暴露的内容

- `思考链` 原文
- `context/policy` 触发器的可配置入口
- 社区、商店、账号相关强入口
- 常驻顶部健康灯和复杂运维指标
- 把所有实现细节直接放入默认界面

## 9. 结论

这一版原型的目标不是“展示系统有多少模块”，而是确保用户能在三个层面上持续建立信任：

1. **知道现在正在发生什么**
2. **知道为什么需要我确认**
3. **知道出了问题如何追溯和恢复**

因此，ClawX 的 UI 主轴应当是：

`工作台 → 对话执行 → 来源/审批/产物 → 记忆/知识/回滚 → v0.2 扩展执行`

而不是：

`把 Agent 平台的所有子系统同时铺开`
