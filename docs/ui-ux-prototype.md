# ZettClaw UI/UX 交互原型

**版本:** 6.0
**日期:** 2026-03-30
**对应 PRD:** v3.0
**对应架构:** v5.0

> Tauri + React + TypeScript 桌面交互范式。
> **核心范式：仿微信三栏布局，每个 Agent 是一个「虚拟人」，支持本地和远程 Agent 社交。左侧图标导航 + 中间 Agent/对话列表 + 右侧内容区。**

---

## 1. 信息架构

### 1.1 整体布局

三栏结构，对标微信桌面版：

```text
ZettClaw
├── A 栏：图标导航栏（56px 固定宽度）
│   ├── 用户头像（顶部）
│   ├── 💬 对话（Agent 对话列表，含本地/远程/群聊）
│   ├── 📇 通讯录（我的 Agent + 远程好友 + Skill 市场）
│   ├── 🌐 广场（Agent 广场，发现公开 Agent）（v0.4）
│   ├── 📚 知识库
│   ├── ⏱ 定时任务（v0.2）
│   ├── 🔗 Connectors（v0.2）
│   └── ⚙️ 设置（底部）
│
├── B 栏：列表面板（280px，随 A 栏选中项切换内容）
│   ├── 搜索栏
│   └── 列表内容（Agent 对话列表 / 通讯录 / 知识源列表 / ...）
│
└── C 栏：主内容区（自适应宽度）
    └── 对话 / Agent 详情 / 知识库检索 / 设置 / ...
```

### 1.2 图标导航栏（A 栏）说明

| 图标 | 名称 | 功能 | 阶段 |
|------|------|------|------|
| 💬 | 对话 | Agent 对话列表（本地/远程/群聊），按最后消息时间排序 | v0.1 |
| 📇 | 通讯录 | 我的 Agent + 远程好友 + Skill 市场 | v0.1 |
| 🌐 | 广场 | 发现和添加公开 Agent，搜索/分类/评分 | v0.4 |
| 📚 | 知识库 | 知识源管理、检索 | v0.1 |
| ⏱ | 定时任务 | 定时/事件触发任务管理 | v0.2 |
| 🔗 | Connectors | 外部渠道接入 | v0.2 |
| ⚙️ | 设置 | 模型 / 安全 / 外观 / 健康 / Relay | v0.1 |

### 1.3 导航显露原则

- v0.1 显示：对话、通讯录、知识库、设置
- v0.2 增加：定时任务、Connectors
- v0.4 增加：广场
- `数据保险箱`：通过对话内操作或设置进入
- `记忆中心`：通过 Agent 详情或对话中查看

### 1.4 全局核心对象

| 对象 | 说明 | 默认入口 |
|------|------|----------|
| 本地 Agent | 用户创建的虚拟分身 | 通讯录「我的 Agent」 |
| 远程 Agent | 别人发布的虚拟人 | 通讯录「远程好友」/ 广场 |
| Conversation | 对话窗口（本地/远程/代理/群聊） | 点击对话列表项 |
| Group | 群聊 | 对话列表 |
| Contact | 远程 Agent 名片缓存 | 通讯录 |
| Artifact | 对话中产生的文件 | Agent 对话页「产出」Tab |
| Memory | Agent 积累的记忆 | Agent 详情页 |
| Source | 知识库文档/资料 | 知识库 |
| Task | 定时/事件驱动任务 | 定时任务页 |

## 2. 页面层级总览

```text
A 栏「对话」 → B 栏 对话列表（本地/远程/群聊混排） → C 栏 对话视图（对话 / 产出 Tab）
A 栏「通讯录」→ B 栏 我的 Agent + 远程好友 + Skill 列表 → C 栏 Agent 详情 / Skill 详情
A 栏「广场」 → B 栏 分类/搜索 → C 栏 Agent 名片详情 + 添加按钮（v0.4）
A 栏「知识库」→ B 栏 知识源列表 → C 栏 检索工作台
A 栏「定时任务」→ B 栏 任务列表 → C 栏 任务详情 / Run 历史
A 栏「Connectors」→ B 栏 渠道列表 → C 栏 渠道配置
A 栏「设置」→ B 栏 设置分类 → C 栏 设置内容
```

**B 栏 — Agent 对话列表**

每个 Agent（本地/远程/群聊）就是一个「聊天对象」，按最后消息时间倒序排列：

| 元素 | 说明 |
|------|------|
| Agent 头像 | 可自定义；远程 Agent 头像右下角有小地球标识；群聊使用拼图头像 |
| Agent 名称 | 如「编程助手」（本地）、「法律顾问@alice」（远程） |
| 来源标识 | 🔵 本地 Agent / 🟢 远程 Agent / 👥 群聊 |
| 最后消息预览 | 单行截断，展示最新一条消息摘要 |
| 时间 | 最后活跃的相对时间 |
| 未读徽标 | 有新消息/待审批时显示红点或数字 |
| 状态指示 | 本地 Agent 运行中绿点；远程 Agent 显示在线状态（online/offline/busy） |
| 加密标识 | 远程对话显示 🔒 端到端加密标识 |

顶部搜索栏支持搜索 Agent 名称和对话内容。右上角 `+` 按钮用于新建 Agent / 添加远程好友 / 创建群聊。

右键菜单：置顶 / 标记已读 / 免打扰 / 进入 Agent 详情 / 删除对话。

**C 栏 — 对话视图**

顶部标题栏：Agent 名称 + 右侧 `···` 更多菜单（Agent 详情 / 切换会话 / 新建会话）+ `▾` 展开该 Agent 详情面板。

对话视图含两个 Tab：

**「对话」Tab**
- 消息流，流式输出
- 执行步骤卡内嵌在消息流中
- 审批请求以消息卡片形式出现

**「产出」Tab**
- 搜索栏 + 筛选
- 该 Agent 的所有历史产出物列表（文档/代码/图片），按时间倒序
- 每条：类型图标 + 标题 + 日期 + 摘要
- 点击预览/下载

**底部输入栏**

| 区域 | 元素 |
|------|------|
| 左侧 | `+` 附件、`Skills` 按钮 |
| 中部 | 文本输入框 |
| 右侧 | 模型选择器（Opus 4 ▾）、发送按钮 |

### 3.1.4 对话中的审批卡片

审批请求以消息卡片形式内嵌在对话流中：

```text
┌─────────────────────────────────────────┐
│ 🔒 编程助手 请求执行 3 个动作             │
├─────────────────────────────────────────┤
│ 1. 写入 src/auth.ts         风险: 中    │
│ 2. 删除 src/old-auth.ts     风险: 高    │
│ 3. 访问 api.github.com      风险: 中    │
├─────────────────────────────────────────┤
│ [查看详情]        [全部拒绝] [逐项审批]  │
└─────────────────────────────────────────┘
```

点击「逐项审批」展开详细审批面板，每项可选：本次允许 / 会话内允许 / 拒绝。

**始终强制确认：** 删除多个文件/目录、系统级动作、白名单外网络访问、用户级共享记忆修改。

**B 栏 — 通讯录列表**

分三个区块：

| 区块 | 说明 |
|------|------|
| 我的 Agent (N) | 本地 Agent 列表，每项显示头像 + 名称 + 模型 + 状态 + 可见性（🌐 公开 / 🔒 私密） |

**B 栏 — 知识源列表**

| 元素 | 说明 |
|------|------|
| 搜索栏 | 按名称模糊匹配 |
| 筛选 Tab | 全部 / 全局 / Agent 专属 |

**C 栏 — 检索工作台**

- 输入问题或关键词，返回命中段落
- 每条结果：标题 + 命中摘要 + 来源 + 「查看原文」
- 默认只展示"命中证据"，高级模式显示分数/chunk ID

**C 栏 — 点击知识源时切换为知识源详情**

- 文档列表（文件名/大小/索引状态）
- 添加/删除文档
- 关联范围设置（全局 / Agent 专属）

**空状态**

```text
还没有添加任何知识源。
添加文档、网页或文件夹，让 Agent 基于你的知识回答问题。
[添加本地文件]  [导入 URL]
```

## 4. 关键交互流程

### 4.1 首次启动引导

```text
启动 → 欢迎页 → 语言与外观 → 本地优先说明（数据留本机/未经允许不联网）
→ 配置模型（本地/云端/测试连接） → 创建首个 Agent → 进入对话页
```

### 4.2 对话请求流程

```text
用户输入 → controlplane → memory recall + knowledge retrieval（并行）
→ 如需工具/写入/联网: 审批卡片插入对话流
→ 执行: 流式输出 + 结构化 Step 卡片
→ DLP 扫描 → 结果完成
```

### 4.3 审批流程

```text
Agent 请求高风险动作 → 审批卡片出现在对话流中 → 用户逐项决定
（本次允许/会话内允许/加入白名单/拒绝） → 记录审计 → 继续/中断
```

### 4.4 导航流程

```text
点击 A 栏图标 → B 栏切换对应列表 → 点击 B 栏条目 → C 栏展示内容
对话页: B 栏选 Agent → C 栏聊天
通讯录: B 栏选 Agent → C 栏详情，点「发消息」跳回对话页
知识库: B 栏选知识源 → C 栏详情/检索
```

## 6. 全局设计规范

### 6.1 导航与布局

| 元素 | 规范 |
|------|------|
| A 栏（图标导航） | 56px 固定，垂直排列图标，顶部用户头像，底部设置 |
| B 栏（列表面板） | 280px 固定，随 A 栏选中项切换内容 |
| C 栏（主内容区） | 自适应宽度，承载对话/详情/设置等 |
| 输入栏 | 对话页 C 栏底部固定，含附件/Skills/文本/模型选择/发送 |

### 6.2 信任与透明

- 不展示 raw chain-of-thought，只展示结构化步骤摘要
- 来源始终可追溯，知识引用能回到原文
- 审批以对话卡片形式出现，自然融入对话流
- 高风险动作后可直接跳审计记录

### 6.3 渐进披露

| 默认态 | 高级态 |
|--------|--------|
| 命中证据摘要 | 检索分数 / chunk ID / 检索方式 |
| 记忆摘要 | JSON 原文 / 访问频次 / 鲜活度 |
| 权限范围 | 能力向量、审计事件 ID |
| 系统状态提示 | CPU / 内存 / 模块状态详情 |

### 6.4 状态反馈

| 状态 | 表达方式 |
|------|----------|
| Agent 空闲 | 对话列表中无特殊标记 |
| Agent 运行中 | 头像旁绿点 + 对话列表最后消息实时更新 |
| 待审批 | 对话列表红点徽标 + 对话内审批卡片 |
| 出错 | 对话内错误消息卡片 + 可恢复操作 |
| 任务完成 | 对话列表更新，显示新消息提示 |

### 6.5 快捷键

| 快捷键 | 动作 |
|--------|------|
| `⌘K` | 全局搜索 / 命令 |
| `⌘N` | 新建对话 |
| `⌘⇧N` | 新建 Agent |
| `⌘1-6` | 切换 A 栏导航项 |
| `⌘.` | 中断当前执行 |

### 6.6 空、载入、错误状态

- **空状态**：必须给下一步动作（如「创建你的第一个 Agent」或「去广场发现 Agent」）
- **载入状态**：骨架屏或步骤进度
- **错误状态**：必须给恢复动作
- **后台处理中**：对话列表中 Agent 头像显示运行动画

### 6.7 视觉规范

专业、克制、可信。深色为主色调。安全/审批用低饱和功能色。标题/正文/辅助/代码四层字体。统一圆角。阴影仅用于浮层。动效重点在消息入场、步骤插入、状态切换。

## 8. 结论

UI 主轴：Agent 社交网络。

`Agent 即虚拟人 → 对话即社交 → 通讯录管理本地+远程 Agent → 广场发现新 Agent → 群聊多 Agent 协作`

用户在四个层面持续建立信任：

1. **知道正在发生什么** — Agent 运行状态、执行步骤实时可见
2. **知道为什么需要确认** — 审批卡片内嵌对话流，上下文清晰
3. **知道出了问题如何追溯** — 通讯录查记忆、设置查审计、知识库查来源
4. **知道通信是安全的** — 远程对话 🔒 端到端加密，Relay 不可读内容
