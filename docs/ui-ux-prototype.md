# ClawX UI/UX 交互原型

**版本:** 3.0
**日期:** 2026-03-17
**对应 PRD:** v2.0
**对应架构:** v4.2

> Tauri + React + TypeScript 桌面交互范式。
> **核心变更：合并「工作台」与「对话」为统一的 Agent Hub，消除入口重复。**

---

## 1. 信息架构

### 1.1 v0.1 导航结构

侧边栏 5 个一级入口（原 6 个，合并工作台与对话）：

```text
ClawX.app
├── 侧边栏
│   ├── 工作台        ← Agent Hub：Agent 列表 + 选中即进对话
│   ├── 记忆中心      ← 用户记忆 / Agent 记忆 / 审计
│   ├── 知识库        ← 知识源管理 / 检索 / 引用
│   ├── 数据保险箱    ← 版本点 / 差异 / 回滚
│   └── 设置          ← 模型 / 安全 / 外观 / 语言 / 健康
│
├── 顶部栏
│   ├── 全局搜索 / 命令面板 (⌘K)
│   ├── 当前上下文标题
│   ├── 服务状态徽标（仅异常时高亮）
│   └── 用户菜单
│
└── 主内容区 + 可选右侧信息面板
```

### 1.2 v0.2 导航扩展

```text
ClawX.app
├── 工作台
├── 记忆中心
├── 知识库
├── 数据保险箱
├── 扩展执行
│   ├── 任务
│   ├── 渠道
│   └── Skills
└── 设置
```

### 1.3 导航显露原则

- `产物管理`：v0.3+ 产物规模增大后才独立成页
- `设备管理`：v0.5 物理 Agent 启用后显露
- `账号 / 备份 / 社区`：始终在设置内

### 1.4 全局核心对象

| 对象 | 用户理解 | 默认入口 |
|------|----------|----------|
| Agent | 长期工作的 AI 角色/能力容器 | 工作台 Agent 列表 |
| Conversation | 一段可持续的对话线程 | 工作台 → 选中 Agent 后 |
| Run | 一次执行过程，可能包含多步 | 对话区 / 任务详情 |
| Memory | 被持久化的偏好、事实、经验 | 记忆中心 |
| Source | 知识源和被引用证据 | 知识库 / 对话右栏 |
| Artifact | Agent 生成的文件或结果 | 对话右栏 / v0.3 产物页 |
| Task | 后台可重复执行的任务定义 | v0.2 扩展执行 |

## 3. 核心页面线框图

### 3.1 工作台（Agent Hub — 首页）

三栏布局：左栏 Agent 列表 → 中栏对话 → 右栏信息面板。

```text
┌────────────────────────────────────────────────────────────────────────────┐

│ 知识库 │            │ [Agent 详情 ▾]                 │                       │
│ 保险箱 │ ● 编程助手  │                               │ 来源                  │

│       │   索引 67%  │ 🤖 我会先检查相关文件，再给出    │ 1. 读取 3 个文件      │
│       │            │ 方案。                         │ 2. 命中 2 份知识源    │
```

**设计要点：**

- Agent 列表始终可见，状态内联（运行中/待审批/索引进度/空闲）
- 点击 Agent 直接展示其最近对话，无需跳转页面
- 会话列表折叠在 Agent 下方，选中 Agent 后展开
- 活跃 Agent 排前面，超过 3 个活跃时显示性能提示

### 3.1.2 创建 Agent 面板

单侧边面板，只收集必填项：

```text
┌──────────────────────────────────────────────┐
│ 创建 Agent                                    │
├──────────────────────────────────────────────┤
│ [模板] [自定义]                               │
│                                               │

│ · Developer   默认能力: write_code / shell    │
│ · Researcher  默认能力: web_search / kb       │
│ · Writer      默认能力: write / edit          │
│                                               │
│ 名称: [编程助手____________]                   │
│ 角色说明: [负责代码编写与重构_______]           │
│ 模型: [Claude Opus 4 ▾]                       │
│ 工作区: [~/.clawx/workspace/dev____]          │

└──────────────────────────────────────────────┘
```

高级配置（System Prompt、能力预设、温度等）在创建后进入 Agent 详情编辑。

### 3.1.3 Agent 管理

Agent 列表 `•••` 菜单：编辑 / 克隆 / 停用 / 删除。

- **克隆**：复制名称（追加"副本"）、模型配置、能力预设、知识关联；不复制对话历史、记忆、运行中任务
- **删除**：二次确认，列出会被删除的内容（配置、专属记忆、对话列表）和会被保留的内容（用户级共享记忆）

### 3.2 对话交互规则

- 默认展开 `来源` 面板
- 检测到多步执行时自动显示 `执行` 面板
- 有待确认动作时 `审批` 面板自动置顶并高亮
- 产物生成后 `产物` 面板显示文件列表
- `记忆` 面板只展示摘要，不展示底层 JSON

### 3.3 对话消息中的执行可视化

在消息流中插入可折叠的结构化步骤卡：

```text
[执行中]
Step 1/4 读取 auth.ts, route.ts, session.ts
结果: 已定位 2 个重复校验逻辑
[查看详情]
```

`查看详情` 展示：输入来源、使用工具、风险说明、结果摘要、相关产物链接。不展示 raw reasoning。

### 3.4 Agent 详情抽屉

从 Agent 列表或对话标题进入，Tabs: 概览 / 配置 / 知识关联 / 记忆概览 / 权限档案(v0.2) / Skills(v0.2)

### 3.5 权限确认弹窗

按影响范围分组的审批单：

```text
┌────────────────────────────────────────────────────────────┐
│ Agent 请求审批                                              │
│ 编程助手 想执行 3 个动作                                     │
├────────────────────────────────────────────────────────────┤
│ 1. 写入工作区文件                                           │
│    范围: ~/.clawx/workspace/dev/src/auth.ts                │
│    风险: 中                                                 │
│    [本次允许] [本会话允许同目录] [拒绝]                      │
│                                                             │
│ 2. 删除旧文件                                               │
│    范围: ~/.clawx/workspace/dev/src/old-auth.ts            │
│    风险: 高                                                 │
│    [本次允许] [拒绝]                                        │
│                                                             │
│ 3. 访问网络                                                 │
│    域名: api.github.com                                     │
│    风险: 中                                                 │
│    [本次允许] [加入白名单后允许] [拒绝]                      │
├────────────────────────────────────────────────────────────┤
│ [查看技术详情]                    [全部拒绝] [按选择继续]    │
└────────────────────────────────────────────────────────────┘
```

**始终强制确认（不允许自动信任覆盖）：** 删除多个文件/目录、T3 Native 系统级动作、白名单外网络访问、用户级共享记忆修改、导出到工作区外路径。

```text

│ 记忆中心                                            🔍 搜索记忆 │
├────────────────────────────────────────────────────────────────┤
│ [全部] [用户记忆] [Agent 记忆] [已固定] [已归档]               │
│                                                                │
│ 已固定                                                         │
│ · 安静的日式餐厅偏好  preference  写作助手  2 天前              │
│                                                                │
│ 全部记忆                                                       │
│ · 高级前端工程师，主用 TypeScript                               │
│ · 周报默认三段式结构                                           │
│                                                                │
│ 右侧详情抽屉: 摘要 / 来源 / 审计记录 / JSON 原文（高级模式）    │
└────────────────────────────────────────────────────────────────┘
```

默认不展示鲜活度算法、访问次数、重要性分数（只在详情抽屉里显示）。

用户级共享记忆修改为高风险动作，需确认卡（展示原内容/新内容/来源/影响范围），修改后写入审计日志。

### 3.7 知识库

左右分栏：知识源列表 + 检索工作台。

- 默认说"命中证据"和"是否可用于回答"，不暴露 RRF/BM25/向量等实现词
- 高级模式才显示检索方式、相关度分数、chunk ID
- 知识源关联范围：`全局`（所有 Agent 可检索）或 `Agent 专属`
- 解除关联不删文件，只改检索可见性

### 3.8 数据保险箱（Vault）

围绕任务级回滚组织：

```text
┌──────────────────────────┬─────────────────────────────────────┐
│ 版本点 / 任务变更集        │ 差异预览                             │
│                          │                                     │
│ 今天 14:30               │ 任务: 重构登录模块                   │
│ 编程助手 · 重构登录模块    │ + src/auth.ts                       │
│ 3 文件 / 12.4 KB         │ ~ src/api.ts                        │
│                          │ - src/old-auth.ts                   │
│ 今天 11:20               │                                     │
│ 编程助手 · 添加 API 端点   │ [还原单文件] [回滚整个任务]          │
│ 1 文件 / 3.2 KB          │ 提示: 回滚前将自动创建当前版本点      │
└──────────────────────────┴─────────────────────────────────────┘
```

### 3.9 扩展执行页（v0.2）

统一容器承载任务/渠道/Skills 三个 Tab。

- **任务**：`time/event` 触发，Run 历史，采纳率/忽略率/负反馈率，抑制与冷却。v0.2 不开放 `context/policy` 触发器
- **渠道**：飞书/Telegram 等，路由到指定 Agent，会话隔离
- **Skills**：已安装列表、执行级别、签名状态、启停

**通知**：不进入一级导航，通过顶部弹出层查看。通知决策四种状态：`send_now` / `send_digest` / `store_only` / `suppress`

### 3.10 设置页

四个主标签：

- **模型**：Provider CRUD、连通性测试、每 Agent 固定模型/智能路由
- **安全**：网络白名单、DLP 状态、审计入口
- **外观与语言**：亮/暗/跟随系统、强调色、中英文
- **健康**：后台服务状态、模块健康、索引占用、错误与恢复建议

底部固定入口：用户协议与隐私政策 / 反馈 / 关于

### 3.11 ⌘K 全局命令面板

支持全局跳转、跨域搜索（对话/知识/记忆）、创建命令、当前页上下文操作。

## 4. 关键交互流程

### 4.1 首次启动引导

```text
启动 → 欢迎页 → 语言与外观 → 本地优先说明（数据留本机/未经允许不联网/工作区内才支持回滚）
→ 选择工作区位置 → 配置模型（本地/云端/测试连接） → 创建首个 Agent → 进入工作台
```

### 4.2 对话请求流程

```text
用户输入 → controlplane → memory recall + knowledge retrieval（并行）
→ 如需工具/写入/联网: Permission Gate → 审批单
→ 执行: 流式输出 + 结构化 Step + 右栏同步更新
→ DLP 扫描 → 结果完成
```

### 4.3 文件工作区边界

- 拖入外部文件 → 复制到 Workspace 受管目录 → Agent 处理受管副本
- 结果默认保存在 Workspace；导出到工作区外需确认，不受 Vault 回滚保护

### 4.4 审批流程

```text
Agent 请求高风险动作 → 按动作分组展示审批单 → 用户逐项决定
（本次允许/本会话允许同范围/加入白名单/拒绝） → 记录审计 → 继续/中断
```

### 4.5 回滚流程

```text
打开保险箱 → 按任务浏览版本点 → 预览差异 → 选择粒度（单文件/文件夹/整个任务）
→ 确认回滚 → 自动创建当前版本点 → 执行回滚
```


## 6. 全局设计规范

### 6.1 导航与布局

| 元素 | 规范 |
|------|------|
| 侧边栏 | v0.1 固定 5 项；v0.2 增加"扩展执行"分组 |
| 顶部栏 | 48px，只保留 ⌘K、当前上下文、状态反馈 |
| 右侧信息面板 | 默认 320px，按数据有无自动显隐 |
| 抽屉 | Agent 详情、记忆详情等优先用右侧抽屉 |
| Modal | 仅用于审批、回滚、删除、创建等需显式确认的动作 |

### 6.2 信任与透明

- 不展示 raw chain-of-thought，只展示结构化步骤摘要
- 来源始终可追溯，知识引用能回到原文
- 审批必须列出路径、域名、范围、风险原因
- 高风险动作后可直接跳审计记录

### 6.3 渐进披露

| 默认态 | 高级态 |
|--------|--------|
| 命中证据摘要 | 检索分数 / chunk ID / 检索方式 |
| 记忆摘要 | JSON 原文 / 访问频次 / 鲜活度 |
| 权限范围 | T1/T2/T3、能力向量、审计事件 ID |
| 系统状态提示 | CPU / 内存 / 模块状态详情 |

### 6.4 状态反馈

| 状态 | 表达方式 |
|------|----------|
| 正常空闲 | 不常驻提醒 |
| 后台忙碌 | 顶栏轻量徽标 + Agent 列表状态 |
| 需审批 | Agent 卡片徽标 + 审批面板高亮 |
| 出错 | 页面内联错误 + 可恢复操作 |
| 成功完成 | Toast + 可跳转结果 |

### 6.5 快捷键

| 快捷键 | 动作 |
|--------|------|
| `⌘K` | 全局搜索 / 命令 |
| `⌘N` | 新建对话 |
| `⌘⇧N` | 新建 Agent |
| `⌘1-5` | 切换一级导航 |
| `⌘.` | 中断当前执行 |

### 6.6 空、载入、错误状态

- **空状态**：必须给下一步动作
- **载入状态**：骨架屏或步骤进度
- **错误状态**：必须给恢复动作
- **后台处理中**：轻状态条或内联卡

### 6.7 视觉规范

专业、克制、可信。浅色工作底，安全/审批用低饱和功能色。标题/正文/辅助/代码四层字体。统一圆角。阴影仅用于浮层。动效重点在面板展开、步骤插入、状态切换。

### 6.8 组件清单

App Shell / Sidebar Nav / Top Command Bar / Agent List Item / Conversation List / Message Bubble / Execution Step Card / Evidence Reference Card / Approval Sheet / Memory Detail Drawer / Knowledge Source Card / Vault Diff Panel / Task Run Row / Notification Popover / Settings Section

## 7. 阶段交付

### 7.1 v0.1 本地闭环

## 8. 结论

UI 主轴：

`选 Agent → 对话执行 → 来源/审批/产物 → 记忆/知识/回滚 → v0.2 扩展执行`
