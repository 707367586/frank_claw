## 全局导航结构

设计稿左侧采用双栏导航：

**图标导航栏（最左侧 A-Bar）**

| 图标 | 功能 | 跳转目标 |
|------|------|---------|
| 聊天气泡 | 对话 | 对话列表/对话窗口 |
| 人物 | 通讯录 | Agent 通讯录 |
| 日历 | 定时任务 | 任务列表 |
| 连接 | 渠道管理 | Connectors |
| 方块 | Agent & Skill | Agent/Skill/模板管理 |
| 齿轮（底部） | 设置 | Settings |

**Agent 列表栏（第二列）**

- `+ New Agent` 按钮
- Agent 搜索框 (`Search agents...`)
- Agent 列表：头像、名称、状态标签（running/空闲/错误）、任务进度条

## 模块 1：Agent 工作台 (Agent Workbench)

### 1.1 Agent 创建（模板方式）

**设计稿:** `New Agent` 页

| 功能点 | 描述 |
|--------|------|
| 模板选项卡 | 顶部 Tab 切换「从模板」和「自定义」 |
| 模板列表 | Developer / Researcher / Writer 三个预设模板，每项含图标、名称、一行描述 |

| Agent 名称 | 输入框，placeholder: `e.g. My Developer Agent` |
| 模型选择 | 下拉框选择绑定模型（如 Claude Opus 4） |
| 创建按钮 | `创建 Agent` 主按钮 + `Cancel` 取消按钮 |

### 1.2 Agent 创建（自定义方式）

**设计稿:** `New Agent (Custom)` 页

| 功能点 | 描述 |

### 1.3 Agent 模板库

### 1.4 Agent 管理列表

**设计稿:** `Agent & Skill` 页（Agent 选项卡）

| 功能点 | 描述 |
|--------|------|

| 功能点 | 描述 |
|--------|------|

## 模块 8：安全执行官 (Security Officer)

### 8.1 权限请求弹窗

**设计稿:** `Permission Modal` 页（ENtBW）

| 功能点 | 描述 |
|--------|------|
| 弹窗标题 | `Agent 请求审批` + 操作来源（如"编辑助手 需执行 3 个动作"） |
| 操作列表 | 编号列表，每项含：操作类型（写入工作文件/删除文件/访问网络）、目标路径/域名、风险等级标签（高危 红色 / 中 黄色） |
| 逐项操作 | 每项有独立的 `允许/拒绝` 按钮组 |
| 详情展开 | 每项可展开查看详细路径、加入白名单选项、读取权限设置 |
| 底部操作 | `全部拒绝` + `按选择结果继续` 按钮 |
| 查看技术详情 | 可选展开技术细节 |

### 8.2 分级执行模型（PRD 定义）

| 层级 | 运行方式 | 典型能力 | 默认权限 |
|------|---------|---------|---------|
| T1: Sandboxed | WASM 沙箱 | 文本转换、JSON 处理 | 默认无文件、无网络 |
| T2: Restricted | 受限子进程 + 工作区隔离 | Shell、Python、浏览器 | 仅授权工作区和命令 |
| T3: Native | 原生宿主能力 | 系统配置、敏感目录 | 每次操作显式确认 |

### 8.3 权限能力模型（PRD 定义）

| 能力 | 格式 | 默认 |
|------|------|------|
| 文件读取 | `fs:read:{path}` | 禁止 |
| 文件写入 | `fs:write:{path}` | 禁止 |
| 网络请求 | `net:http:{domain}` | 禁止 |
| 命令执行 | `exec:shell:{command}` | 禁止 |
| 密钥注入 | `secret:inject:{name}` | 禁止 |

### 8.4 其他安全能力（PRD 定义）

| 功能 | 描述 |
|------|------|
| 网络白名单 | 默认禁止所有出站，GUI 逐一添加 |
| Prompt 注入防御 | 三层：模式匹配 + 内容净化 + LLM 自检 |
| DLP 防泄漏 | 三节点扫描：LLM 出站、工具输出、HTTP 响应 |
| 数据脱敏 | PII 自动脱敏后上云，响应还原 |
| 审计日志 | 追加写入 `~/.clawx/audit/`，链式校验防篡改 |

## 模块 12：记忆中心 (Memory Hub)

**设计稿体现在 Contacts 页记忆区块**

### 12.1 两层记忆架构

| 层级 | 作用域 | 内容 | 权限 |
|------|--------|------|------|
| Agent 记忆 | 单个 Agent | 历史对话摘要、学到的技能、任务日志 | 仅所属 Agent 读写 |
| 用户记忆 | 全局共享 | 姓名、职业、偏好、联系人、术语 | 所有 Agent 可读写 |

### 12.2 结构化存储

每条记忆包含：唯一 ID、类型（事实/偏好/事件/技能）、摘要、详细内容（JSONB）、创建时间、最后访问时间、访问频次、重要性评分（0-10）、来源 Agent ID。

### 12.3 智能遗忘

基于艾宾浩斯遗忘曲线的衰减算法。每条记忆有"鲜活度"分数，随时间衰减，访问时提升。低于阈值自动归档，进一步降低后永久删除。支持"固定"标记永久保留。

### 12.4 验收标准

- 跨 Agent 记忆召回 Top-3 hit rate >= 80%
- 用户可在 GUI 中查看、编辑、冻结或删除任意记忆
- 共享记忆的新增/修改/删除可审计、可回看来源 Agent
